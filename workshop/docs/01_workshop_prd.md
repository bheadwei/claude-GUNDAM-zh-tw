## 1. 教學總覽

| 項目 | 內容 |
| :--- | :--- |
| **課程名稱** | VibeCoding 模板實戰 Workshop：用 Claude Code 跑完一個專案 |

## 2. 教學目標與 OKRs

### 背景與痛點

| 痛點 | 說明 |
| :--- | :--- |
| AI 工具碎片化 | 開發者用 AI 寫程式但缺乏系統化流程，品質不穩定 |
| 文件驅動斷裂 | 需求 → 設計 → 實作之間缺乏可追蹤的連結 |
| 品質把關缺失 | 沒有自動化的 Code Review、測試覆蓋率門檻 |
| 新手不知從何開始 | 模板功能多，但缺乏引導式學習路徑 |

### OKRs

| Objective | Key Result | 衡量方式 |
| :--- | :--- | :--- |
| 學員能獨立使用模板開發專案 | 80% 學員在課後能跑通 `/task-init` → `/task-next` → `/tdd` 完整循環 | 課後問卷 |
| 理解 AI Agent 協作模式 | 學員能說出至少 3 個 Agent 的用途與觸發時機 | 隨堂問答 |
| 建立文件驅動開發觀念 | 學員能解釋 PRD → WBS → 實作的流程邏輯 | 課後作業 |

## 5. 課程大綱與允收標準

### Chapter 1: 認識模板（10 min）

| ID | 內容 | 教學方式 | 允收標準 |
| :--- | :--- | :--- | :--- |
| C1-01 | VibeCoding 模板解決什麼問題 | 講述 + 對比圖 | 學員能說出 3 個痛點 |
| C1-02 | 5 大擴展機制速覽（Hook / Command / Skill / Agent / Context） | 架構圖 | 學員能區分 5 種機制 |
| C1-03 | 目錄結構導覽（.claude/ 下的 7 個關鍵目錄） | 終端機展示 | 學員知道去哪找設定 |
| C1-04 | 環境準備：複製模板 + .mcp.json 設定 | 同步跟做 | 學員本機環境就緒 |

**講師備註：**
- 用「沒有模板 vs 有模板」的對比開場，製造動機
- 展示模板目錄時用 `tree` 指令，但只展開第一層，避免資訊過載
- .mcp.json 設定可提前準備好範例檔，節省時間

### Chapter 2: 專案初始化（15 min）

| ID | 內容 | 教學方式 | 允收標準 |
| :--- | :--- | :--- | :--- |
| C2-01 | 執行 `/task-init`，體驗互動式 Q&A | 講師示範 + 學員跟做 | 成功產出 CLAUDE.md |
| C2-02 | 解說 MVP vs Complete 模式選擇 | 講述 + 比較表 | 學員能判斷何時用哪個 |
| C2-03 | 檢視自動產出的 WBS 任務清單 | 終端機展示 | 學員理解 WBS 結構 |
| C2-04 | `/task-status` 查看初始狀態 | 同步跟做 | 學員看到任務總覽 |

**講師備註：**
- `/task-init` 是最重要的「Aha moment」— 讓學員親手體驗 AI 問問題、自動產出文件
- 刻意在 Q&A 中展示「選推薦選項」和「自訂回答」兩種方式
- 產出 CLAUDE.md 後暫停，打開檔案讓學員看 AI 理解了什麼

**示範專案設定：**
```
專案名稱: TaskFlow
技術棧: Python FastAPI + React + SQLite
開發模式: MVP
功能範圍:
  - 待辦事項 CRUD（新增、完成、刪除）
  - 簡單分類（工作/個人/學習）
  - 本地儲存（SQLite）
  - 極簡 UI（Apple 風格）
```

### Chapter 3: 開發循環 — 核心（25 min）

這是整個 Workshop 的核心，示範 3 輪完整的「取任務 → 規劃 → TDD → 審查」循環。

#### 第 1 輪：後端 API（10 min）

| ID | 內容 | 展示重點 |
| :--- | :--- | :--- |
| C3-01 | `/task-next` 取得第一個任務 | WBS 自動選擇優先任務 |
| C3-02 | `/plan` 建立實作計畫 | **planner agent** 分析依賴、拆解步驟 |
| C3-03 | `/tdd` 測試驅動開發 | **tdd-guide agent** 強制 RED → GREEN → IMPROVE |
| C3-04 | 展示測試從紅變綠的過程 | 強調「先寫測試」的紀律 |
| C3-05 | 打開 FastAPI `/docs` 頁面 | Swagger UI 自動產出，即時測試 API |
| C3-06 | `/review-code` 程式碼審查 | **code-quality-specialist** 抓出問題 |
| C3-07 | git commit | conventional commits 格式 |

**講師備註：**
- 第 1 輪最詳細，每個指令都暫停解說
- `/plan` 產出後停下來，讓學員讀計畫，建立「AI 規劃 → 人類審核」的觀念
- `/tdd` 時刻意讓測試先失敗（RED），截圖/錄影這個紅色輸出，視覺衝擊最強
- FastAPI `/docs` 是額外的 Aha Moment — API 寫完就自動有互動文檔

#### 第 2 輪：前端 UI（10 min）

| ID | 內容 | 展示重點 |
| :--- | :--- | :--- |
| C3-08 | `/task-next` 取下一個任務 | 自動跳到前端任務 |
| C3-09 | `/plan` + `/tdd` 快速帶過 | 學員已熟悉流程，加速 |
| C3-10 | 展示 UI 設計規範自動套用 | 模板 rules/ui-design.md 的效果 |
| C3-11 | 瀏覽器中查看成果 | 啟動 dev server，實際操作 |

**講師備註：**
- 第 2 輪節奏加快，重點放在「重複流程的肌肉記憶」
- 瀏覽器展示是觀眾最興奮的時刻 — 確保 dev server 能順利啟動
- 若時間緊迫，可預先準備好前端骨架，只做增量修改

#### 第 3 輪：修 Bug + 整合（5 min）

| ID | 內容 | 展示重點 |
| :--- | :--- | :--- |
| C3-12 | 刻意引入一個 bug | 建置失敗 |
| C3-13 | `/build-fix` 自動修復 | **build-error-resolver agent** (Haiku) 快速修復 |
| C3-14 | 展示修復前後的 diff | 讓學員看到最小修復原則 |

**講師備註：**
- 這一輪的目的是展示「出錯怎麼辦」— 打消學員的恐懼感
- 刻意犯錯比完美示範更有教學效果

## 6. 示範專案規格 — TaskFlow

### 功能範圍

| 模組 | 功能 | 優先級 | 對應教學章節 |
| :--- | :--- | :--- | :--- |
| **後端 API** | GET/POST/PUT/DELETE /api/todos | Must | Chapter 3 第 1 輪 |
| **資料層** | SQLite + 簡單 schema（id, title, category, done） | Must | Chapter 3 第 1 輪 |
| **前端 UI** | Todo 列表 + 新增表單 + 完成切換 | Must | Chapter 3 第 2 輪 |
| **分類篩選** | 工作/個人/學習 三個分類 tab | Should | Chapter 3 第 2 輪 |
| **深色模式** | 支援系統主題切換 | Could | 若時間允許 |

### 技術選型

| 層級 | 選擇 | 理由 |
| :--- | :--- | :--- |
| 後端語言 | Python 3.11+ | 講師熟悉、語法直覺、學員易讀 |
| 後端框架 | FastAPI | 自帶 OpenAPI 文檔、型別提示、async 支援 |
| 前端 | React + Vite (TypeScript) | 快速啟動，生態成熟 |
| 資料庫 | SQLite (aiosqlite) | 零設定，本機即跑 |
| ORM | SQLModel | FastAPI 作者同生態，Pydantic + SQLAlchemy 整合 |
| 後端測試 | pytest + httpx (AsyncClient) | Python 測試標準、支援 async |
| 前端測試 | Vitest + Playwright | 與 Vite 生態整合 |
| 樣式 | Tailwind CSS | 與模板 UI 規範契合 |
| API 文檔 | FastAPI 自動產出 Swagger UI | 教學中可直接在 /docs 頁面示範 API |

### 明確不做

- 使用者認證/授權
- 雲端部署
- 即時同步（WebSocket）
- 拖拉排序
- 複雜的狀態管理（Redux 等）

## 8. 教學風險與對策

| 風險 | 可能性 | 影響 | 對策 |
| :--- | :--- | :--- | :--- |
| API Key 過期或額度不足 | 中 | 高 | 預備離線錄影作為 fallback |
| 學員環境差異大（Windows/Mac/Linux） | 高 | 中 | 提供 Docker 環境 or 預錄影片對照 |
| MCP Server 連線失敗 | 中 | 中 | 只依賴核心功能，MCP 作為加分項展示 |
| `/task-init` 產出不如預期 | 低 | 中 | 準備好一份「標準答案」CLAUDE.md 可直接替換 |
| 單堂時間不夠 | 中 | 中 | Chapter 5 可彈性縮減，優先保 Chapter 2-3 |

## 9. 課後延伸

### 作業建議（學員自行練習）

1. **初級：** 用模板建立一個「記帳本」App，跑完 `/task-init` → `/task-next` → `/tdd` 循環
2. **中級：** 為 TaskFlow 加上「到期日」欄位，練習 SQLModel migration + 前端更新
3. **進階：** 為 TaskFlow 加上使用者認證（FastAPI Security + JWT），體驗安全相關 Agent 協作

### 推薦學習路徑

```
Workshop 完成
    │
    ├── 想深入 Agent 機制 → 閱讀 .claude/agents/ 下的 13 個定義檔
    ├── 想客製化規則    → 閱讀 .claude/rules/ 並修改或新增
    ├── 想擴充 Skills   → 瀏覽 custom-rule&skill/skills/ 的 94+ 選項
    ├── 想做完整專案文件 → 使用 /project-docs 走 VibeCoding 16 模板
    └── 想建 MCP Server → 使用 mcp-builder skill
```
