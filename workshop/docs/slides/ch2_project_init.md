## Slide 2: 一切從 /task-init 開始

```
┌─────────────────────────────────────────┐
│                                         │
│  你輸入：  /task-init TaskFlow          │
│                                         │
│  AI 會做：                              │
│                                         │
│  1. 問你一系列問題（互動式 Q&A）         │
│     - 專案做什麼？                      │
│     - 用什麼技術？                      │
│     - 範圍多大？                        │
│     - MVP 還是完整模式？                │
│                                         │
│  2. 根據你的回答，自動產出：             │
│     - CLAUDE.md   （專案說明書）         │
│     - wbs.md      （任務拆解）          │
│                                         │
│  3. 從此 AI 知道你的專案脈絡             │
│                                         │
└─────────────────────────────────────────┘
```

**講師口述：** 這是模板最關鍵的第一步。AI 不會直接開始寫程式，而是先問你問題、理解需求，再產出文件。這和我們日常開發的「先開會討論需求」是一樣的邏輯。

## Slide 3: MVP vs Complete 模式

```
┌──────────────────┬──────────────────────┐
│     MVP 模式      │    Complete 模式      │
├──────────────────┼──────────────────────┤
│ 輕量 Tech Spec    │ 完整 16 份文件        │
│ 快速驗證想法      │ 每個決策有文件記錄     │
│ 1-2 人小專案      │ 3+ 人團隊專案         │
│ 原型 / POC        │ 正式產品 / 合規需求   │
│                   │                      │
│ 今天用這個 ←      │ 團隊大專案再升級      │
└──────────────────┴──────────────────────┘

MVP 可以隨時升級為 Complete
```

**講師口述：** 今天教學用 MVP 模式，70 分鐘跑得完。實際工作中如果是正式產品，可以選 Complete 模式，會走完 16 份文件模板。MVP 隨時可以升級，不用重來。

## Slide 4: 產出物解說 — CLAUDE.md

```
CLAUDE.md = 專案的「身分證」
─────────────────────────────

AI 每次對話都會讀這份檔案，所以它：

• 知道專案用什麼技術棧
• 知道程式碼規範（命名、風格）
• 知道目錄結構
• 知道測試策略
• 知道部署方式

→ 你不需要每次都重新解釋
→ 團隊成員接手也能立刻上手
```

**講師口述：** CLAUDE.md 是整個模板的核心。想像它是你的專案 wiki，但 AI 每次都會自動讀取。

---

## Slide 5: 產出物解說 — WBS (Work Breakdown Structure)

```
WBS = 工作拆解結構
─────────────────────

1.0 專案管理
├── 1.1 需求分析        ← /task-init 產出
└── 1.2 架構設計

2.0 後端開發
├── 2.1 基礎架構        ← /task-next 會指派
├── 2.2 CRUD API
└── 2.3 篩選功能

3.0 前端開發
├── 3.1 元件開發
└── 3.2 互動功能

4.0 測試
└── 4.1 E2E 測試

每個任務有：編號 | 負責人 | 狀態 | 工時
```

**講師口述：** WBS 就是你的待辦清單，但是 AI 自動幫你拆好的。之後用 `/task-next` 就會從裡面挑最該做的任務給你。

## Slide 6: 現場操作 — /task-init

```
⚡ 現在一起做

1. 打開終端機
2. cd 到你的工作目錄
3. 輸入：claude
4. 輸入：/task-init TaskFlow

跟著 AI 的問題回答：
• 專案描述：極簡待辦清單應用
• 技術棧：Python FastAPI + React + SQLite
• 模式：MVP
• 功能：CRUD + 分類篩選
```

**講師口述：** 請大家跟著我一起做。注意看 AI 問問題的方式——它每次只問一題，等你回答完才問下一題。
