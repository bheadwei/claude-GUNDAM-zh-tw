# 互動問答規則（Interactive Q&A）

所有需要向使用者收集資訊、確認選項或做決策的流程，**必須**遵守以下規則。

## 規則

1. **使用 `AskUserQuestion` 工具** — 禁止在純文字中列出多題或選項
2. **一次一題** — 每次只問 1 題（最多 2 題且高度相關），收到答案再決定下一題
3. **提供 2-4 個預設選項** — 推薦選項放第一個並標註 `(Recommended)`，每個選項寫清楚 description
4. **記錄問答歷史** — 所有問答寫入 `.claude/qa-history/YYYY-MM-DD-HHMMSS-<command-name>.md`
   - 先在記憶中收集所有問答，流程結束後**一次性 Write 整份檔案**（省 token）
   - 格式：每題含選項、答案、時間；結尾寫最終結論

## 例外（不需走 Q&A）

- 純資訊顯示（`/task-status`、`/time-log`）
- 唯一選項，無需使用者決定
- 使用者在指令參數中已明確指定
