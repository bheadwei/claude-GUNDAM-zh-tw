# 深度研究

使用 firecrawl 和 exa MCP 工具，從多個網路來源產出有引用來源的研究報告。

## 啟動時機

- 使用者要求深入研究任何主題
- 競品分析、技術評估或市場規模估算
- 對公司、投資者或技術進行盡職調查
- 任何需要綜合多個來源的問題
- 使用者說「研究」、「深入調查」、「調研」或「目前 X 的現況如何」

## MCP 需求

至少需要其中一個：
- **firecrawl** — `firecrawl_search`、`firecrawl_scrape`、`firecrawl_crawl`
- **exa** — `web_search_exa`、`web_search_advanced_exa`、`crawling_exa`

兩者搭配使用覆蓋範圍最廣。在 `~/.claude.json` 或 `~/.codex/config.toml` 中設定。

## 工作流程

### 步驟 1：理解目標

問 1-2 個快速釐清問題：
- 「你的目標是學習、做決策，還是撰寫文件？」
- 「有特定的角度或深度要求嗎？」

如果使用者說「直接研究就好」— 跳過，以合理預設值進行。

### 步驟 2：規劃研究

將主題拆為 3-5 個子問題。範例：
- 主題：「AI 對醫療的影響」
  - 目前 AI 在醫療領域的主要應用？
  - 已測量的臨床成效？
  - 法規挑戰為何？
  - 領先的公司有哪些？
  - 市場規模和成長趨勢？

### 步驟 3：多源搜尋

對每個子問題，使用可用的 MCP 工具搜尋：

**使用 firecrawl：**
```
firecrawl_search(query: "<子問題關鍵字>", limit: 8)
```

**使用 exa：**
```
web_search_exa(query: "<子問題關鍵字>", numResults: 8)
web_search_advanced_exa(query: "<關鍵字>", numResults: 5, startPublishedDate: "2025-01-01")
```

**搜尋策略：**
- 每個子問題使用 2-3 種不同關鍵字組合
- 混合一般搜尋和新聞搜尋
- 目標收集 15-30 個獨立來源
- 優先順序：學術 > 官方 > 知名新聞 > 部落格 > 論壇

### 步驟 4：深讀關鍵來源

對最有價值的 URL，抓取完整內容：

**使用 firecrawl：**
```
firecrawl_scrape(url: "<url>")
```

**使用 exa：**
```
crawling_exa(url: "<url>", tokensNum: 5000)
```

深讀 3-5 個關鍵來源，不要只依賴搜尋摘要。

### 步驟 5：綜合撰寫報告

報告結構：

```markdown
# [主題]：研究報告
*產出日期：[日期] | 來源數：[N] | 信心度：[高/中/低]*

## 摘要
[3-5 句關鍵發現概述]

## 1. [第一個主題]
[發現與行內引用]
- 要點 ([來源名稱](url))
- 佐證資料 ([來源名稱](url))

## 2. [第二個主題]
...

## 重點結論
- [可行動洞察 1]
- [可行動洞察 2]
- [可行動洞察 3]

## 來源
1. [標題](url) — [一句話摘要]
2. ...

## 方法論
搜尋 [N] 組查詢，跨網頁與新聞。分析 [M] 個來源。
調查的子問題：[清單]
```

### 步驟 6：交付

- **短篇主題**：直接在對話中貼完整報告
- **長篇報告**：對話中貼摘要 + 重點結論，完整報告存檔

## 平行研究（使用子 Agent）

對於廣泛主題，使用 Claude Code 的 Task 工具平行處理：

```
同時啟動 3 個研究 agent：
1. Agent 1：研究子問題 1-2
2. Agent 2：研究子問題 3-4
3. Agent 3：研究子問題 5 + 跨領域主題
```

每個 agent 搜尋、閱讀來源並回傳發現。主 session 綜合為最終報告。

## 品質規則

1. **每個論述都需要來源。** 不可有無出處的斷言。
2. **交叉驗證。** 若只有一個來源提到，標記為「未經驗證」。
3. **時效性重要。** 優先使用近 12 個月的來源。
4. **承認空白。** 若找不到某子問題的好資料，直接說明。
5. **不可臆造。** 不知道就說「找不到足夠資料」。
6. **區分事實與推論。** 明確標示估計、預測和觀點。
